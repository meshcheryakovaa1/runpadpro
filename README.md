# RunPad Pro

Комплексное решение для администрирования компьютерных классов и игровых залов.

## Версия

**RunPad Pro v2.25**

## Описание

**RunPad Pro** — это профессиональная система управления компьютерными классами, предназначенная для использования в:
- Школах и университетах
- Интернет-кафе
- Библиотеках
- Офисах
- Игровых залах
- Других учреждениях с множественными рабочими станциями

Программа обеспечивает централизованное управление клиентскими компьютерами Windows, включая:
- Мониторинг состояния рабочих станций
- Удаленное управление
- Систему восстановления **Rollback** (автоматическое восстановление системы)
- Защиту от несанкционированного доступа
- Управление доступом пользователей

## Структура проекта

```
runpadpro/
├── admin/          # Административные утилиты
│   ├── rfmclient/     # Клиент удаленного файлового менеджера
│   ├── rsdbbackup/    # Резервное копирование БД
│   ├── rsdbconf/      # Конфигурация БД
│   ├── rsdbview/      # Просмотр БД
│   ├── rsrdclient/    # Клиент удаленного рабочего стола
│   ├── rssettings/    # Настройки системы
│   └── ...
├── client/         # Клиентская часть (32+ модулей)
│   ├── rshell/        # Оболочка клиента
│   ├── rsbrowser/     # Браузер
│   ├── bodyXXX/       # Различные модули функциональности
│   ├── rsblock/       # Блокировка
│   ├── rsrdserver/    # Сервер удаленного рабочего стола
│   └── ...
├── server/         # Серверная часть
│   ├── rs_server/     # Основной сервер
│   └── rs_server_setup/ # Установщик сервера
├── operator/       # Модуль оператора
│   ├── rsoperator/    # Консоль оператора
│   └── rsstat/        # Статистика
├── common/         # Общие компоненты
│   ├── help/          # Документация (CHM)
│   ├── redist/        # Распространяемые компоненты
│   └── setup_selector/ # Выбор установки
└── tools/          # Вспомогательные инструменты
```

## Основные компоненты

### Server
Серверная часть системы, обеспечивающая:
- Централизованное управление клиентами
- Хранение конфигураций
- Координацию работы системы

### Client
Клиентская часть, включающая:
- Оболочку рабочего стола (rshell)
- Модули блокировки и контроля
- Интеграцию с приложениями
- Систему восстановления Rollback
- 30+ специализированных модулей (body*)

### Operator
Консоль оператора для:
- Мониторинга состояния клиентов
- Управления сеансами
- Просмотра статистики

### Admin
Административные инструменты:
- Удаленное управление файлами (rfmclient)
- Конфигурация базы данных (rsdbconf)
- Резервное копирование (rsdbbackup)
- Удаленный рабочий стол (rsrdclient)
- Настройка системы (rssettings)

## Сборка проекта

### Полная сборка всех компонентов

```batch
make.bat
```

Скрипт последовательно собирает все модули:
1. Admin
2. Client
3. Operator
4. Server
5. Common (должен собираться последним!)

### Создание установочных пакетов

```batch
install.bat
```

Создает установочные пакеты для всех компонентов в директории `!out`.

## Системные требования

- **ОС**: Windows (все версии с поддержкой Win32 API)
- **Среда разработки**: Visual C++ (судя по .cpp/.h файлам)
- **Дополнительно**:
  - Windows SDK
  - Компилятор ресурсов для .rc файлов

## Технологии

- **Язык**: C/C++, JavaScript (для UI тем)
- **GUI Framework**: Win32 API, MFC
- **Сеть**: Сокеты Windows
- **База данных**: Встроенная БД (rsdb*)
- **Компиляция**: Batch-скрипты для автоматизации сборки

## Разработка

Проект поддерживает модульную структуру, где каждый компонент собирается независимо. При добавлении новых функций убедитесь, что:

1. Модуль корректно интегрируется с существующей архитектурой
2. Обновлены соответствующие make.bat скрипты
3. Соблюдается порядок сборки (common всегда последним)
4. Версия обновлена в `common/version.inc`
